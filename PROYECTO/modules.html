Módulos

Antes de ES6, se usaban librerías como Browserify para crear módulos en el lado del cliente, y require en Node.js. Con ES6, ahora se pueden usar directamente módulos de todo tipo (AMD y CommonJs).

Exportando en CommonJs

          
            module.exports = 1;
            module.exports = { foo: 'bar' };
            module.exports = ['foo', 'bar'];
            module.exports = function bar () {};

Exportando en ES6

En ES6, se puede exportar de varias maneras. Se pueden realizar exportaciones nombradas:

          
            export let name = 'David';
            export let age  = 25;​​

Así como exportar una lista de objetos:

          
            function sumTwo(a, b) {
              return a + b;
            }
            function sumThree(a, b, c) {
              return a + b + c;
            }
            export { sumTwo, sumThree };

Tambíen se pueden exportar funciones, objetos y valores (etc.) simplemente usando la palabra clave export:

          
            export function sumTwo(a, b) {
              return a + b;
            }
            export function sumThree(a, b, c) {
              return a + b + c;
            }

Finalmente, se pueden exportar enlaces por defecto:

          
            function sumTwo(a, b) {
              return a + b;
            }
            function sumThree(a, b, c) {
              return a + b + c;
            }
            let api = {
              sumTwo,
              sumThree
            };
            export default api;
            /* Que es lo mismo que
            * export { api as default };
            */

Mejores Prácticas

Siempre usar el método export default al final del módulo. Hace claro qué está siendo exportado, y ahorra tiempo buscando qué nombre se puso a cada valor exportado. Además, una práctica común en módulos CommonJS es exportar un solo valor u objeto. Siguiendo este paradigma, el código se vuelve fácilmente legible y se puede interpolar entre módulos CommonJS y ES6.

Importando en ES6

ES6 provee varias formas de importar. Se puede importar un archivo entero:

          
            import 'underscore';

Nota: simplemente importar un archivo entero ejecuta todo el código al nivel más alto del archivo.

De manera similar a Python, se tienen importaciones nombradas:

          
            import { sumTwo, sumThree } from 'math/addition';

También se pueden renombrar estas importaciones:

          
            import {
              sumTwo as addTwoNumbers,
              sumThree as sumThreeNumbers
            } from 'math/addition';

Además, se pueden importar todas las cosas (namespace import):

          
            import * as util from 'math/addition';

Finalmente, se puede importar una lista de valores desde un módulo:

          
            import * as additionUtil from 'math/addition';
            const { sumTwo, sumThree } = additionUtil;

Importando desde el enlace por defecto de la siguiente manera:

          
            import api from 'math/addition';
            // Same as: import { default as api } from 'math/addition';

Mientras es mejor mantener las exportaciones simples, se pueden a veces mezclar importaciones por defecto e importaciones mezcladas si es necesario. Cuando se está exportando de la siguiente manera:

          
            // foos.js
            export { foo as default, foo1, foo2 };

Se pueden importar como:

          
            import foo, { foo1, foo2 } from 'foos';

Cuando se importa un módulo exportado usando sintaxis commonjs (como React), se puede hacer:

          
            import React from 'react';
            const { Component, PropTypes } = React;

Y se puede simplificar más, usando:

          
            import React, { Component, PropTypes } from 'react';

Nota: valores exportados son bindings, no referencias. Por lo tanto, al cambiar el binding de una variable en un módulo se afecta el valor dentro del módulo exportado. Evitar cambiar la interfaz pública de estos valores exportados.
